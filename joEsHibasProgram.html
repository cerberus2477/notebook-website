Jó és hibás programok

Mi a célja egy programnak?

Egy program célja, hogy egyszerűbbé, gyorsabbá és kényelmesebbé tegyen valamilyen tevékenységet, megoldjon egy problémát amivel általában a hétköznapi életünkben találkozunk.

Jó program szempontjai fejlesztői szemszögből:

  • karbantarthatóság, bővíthetőség (áttekinhető legyen, hibákat egyszerűen lehessen látni, ha akarjuk lehet plusz dolgokat hozzáadni és nem kell átírnunk az egészet)
  • újrafelhasználhatóság: (egymáshoz hasonló feladatoknak egyes részeit lehessen ugyanúgy felhasználni és azt csak kiegészíteni vagy részleteket módosítani, ne újraírni)
  • kompatibilitás (más programmal összeépíthető)
  • ellenőrizhetőség (könnyen előállítható tesztelési eljárások)
  • integritás (rendszerhibák nem teszik tönkre a programot, nem teszik használhatatlanná)
Egy példa rossz kódra:
namespace ConsoleApp11{internal class Program{static void Main(string[] args){int j = 80;for(int i = 9999; i > 9980;i=i-1) { Console.WriteLine("Hello, World!");j = Convert.ToInt32( Console.ReadLine());}}}}}
Egy példa a jó kódra
namespace joKod { internal class Program { static void Main(string[] args) { bool valid = false; int Number; while (valid== false) { string Input = Console.ReadLine(); if (int.TryParse(Input, out Number)){ valid = true; } else { Console.WriteLine("Kérem, egy egész számot írjon be."); } } } } }

Az első kóddal több probléma is adódik, legszembetűnőbb az átláthatatlansága, ezt azzal lehet elkerülni hogy a kódot nem egy sorba írjuk, a for ciklust nem egy indokolatlanul nagy számtól visszafelé indítjuk, valamint az int változók alaérténkét(ha más funciójuk nincs) 0-ás értéket kapnak. Második probléma az, hogy nem juttat információt a felhasználónak és nem is ellenőrzi megfelelő adattipust ad-e meg. Harmadik probléma a fölösleges iteráció hiszen csak töbször bekéri majd feleslegesen átírja az adatot

A második kód Ez a kód átlátható, ellenőrzi hogy jó adatot adott-e meg a felhasználó, ha nem újrakéri. Minden adatot felhasznál benne, más kódokba remekül beilleszthető és bővíthető.
